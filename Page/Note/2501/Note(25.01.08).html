DB Cursor

DB Cursorとは、DBのQuery結果集合から、特定Rowを指すPointerである。データを１Rowずつ処理できるようにする。 大量のデータを処理するときにより精密な操作が可能だけど、性能面で遅く、リソースを多く要るので単純作業では向いていない。

Client-Side/Server-Side Cursor

分かりやすく比較すると、どっちがメモリの負担を持つかの違いだと思う。

Client-Side Cursor

Client ApplicationがDB ServerからQuery結果を全て取得し、処理を行う。

Serverへの負荷が少なく、Client側でデータを自由に操作可能。だが、大量データ処理時にClientのメモリやネットワーク帯域幅を多く消費する。


Server-Side Cursor

DB ServerがCursorを管理し、Clientは結果セットを一度に全て取得するのではなく、必要な分だけ順次取得する。

大量データ処理時のメモリ使用量を抑え、ネットワークコストを削減可能。だが、Serverの負荷が増加し、ClientとServer間の通信が頻繁になる可能性がある。
