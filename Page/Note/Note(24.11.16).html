버블 정렬 문제

백준 2750번  문제가 쉽긴 한데, 실행 서너 번 돌려서야 맞췄다. 그래도 이런 문제는 금방 풀 수 있어서 다행인 듯. 조건문을 써서, 이미 배열이 정렬된 경우(내부 반복문에서 스왑이 일어나지 않은 경우) 상위 반복문을 탈출하는 조건문도 잘 추가했다.

스택 문자열 문제

백준 4949번  스택 문자열 문제에 약한 것 같아서, 계속 보충하고자 또 이 문제를 풀기로 했다. 2시간 걸렸다.... 스택 자료구조가 어떤 놈인지는 알겠는데, 그 스택을 문제에 놓인 상황에서 잘 활용하는 방법을 잘 모르는 것 같다. 더군다나, 기초적인 구현력조차 많이 딸리는 거 같다. 반복문과 조건문을 똑바로 쓰는 방법을 잘 모르는 듯.... 정진하자.

어제 말한 대로 The Space에 TinyMCE를 적용했다.

값 바인딩 하고, 스킨등 툴바 플러그인 좀 만지고, post 페이지는 좀 더 디자인적으로는 손 봐야겠지만, 기능적으로는 됐다 싶다.
WYSIWYG 에디터에서 html문으로 텍스트를 저장하기 때문에 read 페이지에서 html 렌더링 해서 보여주도록 했다.
다만, v-html과 같은 기본 옵션을 사용하려고 했는데, xss(Cross-site Scripting; 대충, 스크립트 코드를 삽입하여 페이지 본연의 기능 외의 동작을 수행하게 하는 공격법) 보안 문제로 오류가 떠서, vue-dompurify-html을 적용했다. dompurify를 사용하므로써, 악의적인 스크립트 코드는 실행되지 않도록 필터링해서 출력하게 된다고 한다.
